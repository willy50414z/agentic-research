## 審查步驟

依照上方的審查規則，對待審查的 spec 進行完整審查：

1. **識別研究領域**
   若為量化交易策略，繼續執行 2～5；若無法判定，列為待釐清問題。

2. **填寫已知欄位**
   將 user 明確寫出的內容完整保留，原文呈現，不改變意圖。

3. **補全缺漏欄位**
   以上方範例 spec 為格式基準，逐一核對必填欄位：
   - 市場論點
   - 交易範圍（標的、資產類別、交易所、方向、時間週期、交易時段）
   - 資料規格（來源、必要欄位、訓練／驗證／測試期間、清理規則）
   - 進場訊號（精確條件：指標數值、交叉、閾值）
   - 出場訊號（精確條件 + 停損）
   - 倉位 Sizing 與風險規則
   - 執行假設（訂單類型、滑價、手續費、延遲、再平衡頻率）
   - 績效門檻（min win rate、max drawdown、min alpha ratio、profit factor IS/OOS）
   - Plugin（固定填入 `quant_alpha`）

   可合理推斷的缺漏欄位直接填入並加註 *(假設)*，在 `## 假設說明` 條列原因。
   禁止使用模糊詞：「適當」、「最佳化」、「TBD」、「視情況而定」。

4. **檢查回測標準符合性**
   確認進出場訊號、績效門檻、執行假設是否精確且可量化。
   若有欄位模糊、矛盾、或缺少無法合理推斷的關鍵商業決策，列為待釐清問題。

5. **依照範例 spec 的格式撰寫 `reviewed_spec_initial.md`**
